SOLO usa esta función si:

  • Tienes un backup reciente y válido
  • Estás seguro de que quieres perder todos los datos actuales
  • Has hecho un backup de los datos actuales antes de proceder
  • Entiendes completamente las consecuencias
Recomendación: Antes de restaurar, descarga un backup completo de la base de datos actual usando la opción "Descargar Base de Datos Completa" en el menú de administración.

Información sobre la Restauración
Proceso de restauración:
  1. Se creará un backup automático del archivo actual (solo SQLite)
  2. Se reemplazará completamente la base de datos con el archivo seleccionado
  3. Se reiniciarán las conexiones de la base de datos
  4. La aplicación continuará funcionando con los datos restaurados
Formatos soportados:
  • SQLite (.db): Archivo binario de SQLite
  • PostgreSQL (.sql): Dump SQL generado con pg_dump
Consejo: Si tienes dudas, contacta con el administrador del sistema antes de proceder.
